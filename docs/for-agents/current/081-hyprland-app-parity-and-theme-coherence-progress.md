### Slice 6 ÔÇö Waybar tray icon color coherence & UI grouping

**Problem**: Tray icons (blueman, nm-applet, udiskie) rendered as full-color SVGs from `16x16/panel/`
instead of tinted symbolic SVGs. Additionally, waybar had an issue where GTK menu interaction triggered
a state reset causing the icon to drop its CSS tinting and fallback to `#dfdfdf` hardcoded color.

**Fix**: 
- Replaced `catppuccin.gtk.icon.enable = true` logic with a custom `papirus-tray-patched` derivation
  in `modules/features/desktop/_theme-catalog.nix`.
- This derivation intercepts the symlinked source SVGs from `catppuccin-papirus-folders`, resolves their
  targets, and directly `sed`s `#dfdfdf` and `#444444` to the `@lavender` hex color (`#b4befe`).
- Waybar now renders the correctly tinted icon statically and it perfectly survives `PRELIGHT` state
  resets generated by GTK popup menus.
- Grouped `cpu`, `memory`, and `temperature` inside a `group/hardware` block.
- Implemented `custom/separator` visually resembling `|` (with slight opacity) spacing elements correctly.

Validation:
- Checked waybar log during tray icon interaction; no white fallback Ôťů
- Configured waybar modules grouped inside `#hardware` CSS selector correctly. Ôťů
